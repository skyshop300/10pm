기존에 구현했던 방식대로 로그 추적기를 구현하면 기존코드를 많이 수정해야한다.
이러한 한계를 극복하기 위해 다음의 요구사항을 추가한다.
- 원본 코드를 전혀 수정하지 않는 로그추적기 구현.
- 특정 메서드는 로그를 출력하지 않는 기능
  - 보안상 일부는 로그를 출력이 되면 안된다.
- 다음과 같은 다양한 케이스에 적용할 수 있어야한다.
  - V1: 인터페이스가 있는 구현 클래스
  - V2: 인터페이스가 없는 구현 클래스
  - V3: Component Scan 대상에 기능 적용
- 이를 구현하기 위해 Proxy Pattern을 적용한다.
